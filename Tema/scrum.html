Introducción

Este documento describe la implementación de Scrum en el proyecto de Terminal de Buses. El objetivo de este proyecto es desarrollar un sistema para la facturación de boletos, para la terminal de buses ubicada en la ciudad de La Paz, que permita la agilización del proceso de compra.

Product Backlog

  • Registro de usuarios: Implementar un sistema de registro de usuarios para almacenar la información personal de los pasajeros y agilizar el proceso de compra.
  • Gestión de precios: Mantener los precios de los boletos de un viaje, evitando que existan cobros extra en ultimo momento.
  • Generación automática de facturas: Desarrollar un sistema que genere automáticamente facturas electrónicas para cada compra realizada, mejorando la eficiencia y reduciendo el tiempo de atención al cliente.
  • Gestión de histórico de ventas: Implementar un módulo que permita almacenar y consultar el historial de ventas, facilitando la identificación de patrones y análisis de datos para la toma de decisiones.
  • Generación de reportes de ventas: Implementar un sistema de generación automática de informes y reportes de ventas para facilitar la toma de decisiones estratégicas.

Sprint

Sprint 1. - Recopilación de Información

El sprint de Recolpilación de Información tendra una duracion de 1 semana.
Se debran cumplir los siguientes objetivos:

  • Definir los objetivos y alcance del proyecto en base a la terminal de buses de La Paz.
  • Realizar investigaciones y análisis del dominio del sistema, comprendiendo su funcionamiento, procesos y requisitos específicos.
  • Diseñar y aplicar encuestas a pasajeros de la terminal de buses de La Paz.
  • Recopilar y analizar los resultados de las encuestas y entrevistas, identificando patrones, necesidades y requerimientos prioritarios.
  • Actualizar la documentación del proyecto, incluyendo el acta de inicio y el plan de gestión de requisitos.

Sprint 2. - Diagrama de Contexto

El sprint de Diagrama de Contexto tendra una duracion de 3 semanas.
Se debran cumplir los siguientes objetivos:

  • Revisar la documentación recopilada durante el sprint Recolpilación de Información para comprender el sistema y sus interacciones con el entorno.
  • Diseñar y crear el diagrama de contexto, que muestre las interacciones entre el sistema y sus actores externos.
  • Actualizar la documentación del proyecto, incluyendo el acta de inicio y el plan de gestión de requerimientos, con la inclusión del diagrama de contexto.

Sprint 3. - Diagrama de Clases

El sprint de Diagrama de Clases tendra una duracion de 2 semanas.
Se debran cumplir los siguientes objetivos:

  • Identificar las entidades principales del sistema y sus atributos, así como las relaciones entre ellas.
  • Realizar investigaciones y análisis del dominio del sistema, comprendiendo su funcionamiento, procesos y requisitos específicos.
  • Definir las clases necesarias para representar las entidades y sus interacciones en el sistema.
  • Crear el diagrama de clases utilizando una herramienta de modelado, asegurando su correcta estructura y notación.
  • Actualizar la documentación del proyecto, incluyendo el documento de diseño y las especificaciones técnicas, con la inclusión del diagrama de clases.

Practicas Utilizadas

  • Planificación de sprint: Antes de cada sprint, el equipo lleva a cabo una reunión de planificación en la que se seleccionan las historias de usuario y se definen las tareas a realizar durante el sprint.
  • Revisiones de sprint: Al final de cada sprint, el equipo realiza una reunión de revisión en la que el equipo muestra el trabajo completado al Product Owner. Esta reunión es una oportunidad para recibir retroalimentación, hacer demostraciones y ajustar las prioridades del producto.
  • Retrospectivas de sprint: Después de cada sprint, el equipo lleva a cabo una retrospectiva para evaluar su desempeño y discutir mejoras, identifican aspectos positivos, desafíos y acciones de mejora para el próximo sprint.
  • Colaboración y comunicación constante: Scrum promueve la colaboración y la comunicación efectiva entre todos los miembros del equipo. Esto incluye compartir conocimientos, trabajar en estrecha colaboración en tareas y tomar decisiones conjuntas para optimizar el proceso de desarrollo.
