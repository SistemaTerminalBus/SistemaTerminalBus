ISO 25010

El presente modelo de calidad determina las características de calidad que se van a tener en cuenta a la hora de evaluar las propiedades de un producto software determinado. La ISO 25010/IEC se compone por ocho características de calidad:

  • Funcionalidad.- El sistema satisface las necesidades haciendo unicamente los precedimientos necesarios y adecuados.
    Se realizan Pruebas de Regresión, Pruebas del Sistema, Pruebas Unitarias, Pruebas de Procedimientos, etc.
  • Fiabilidad.- El sistema o componente desempeña las funciones especificadas, cuando se usa bajo unas condiciones y periodo de tiempo determinados. Sus conceptos son: la madurez, disponibilidad, capacidad de recuperación y tolerancia a fallas.
    Se realizan pruebas de respaldo, recuperación y recuperación de desastres.
  • Usabilidad.- En sistema debe cumplir Reconocibilidad de la adecuación, Aprendizabilidad, Operabilidad, Protección contra errores de usuario, Estética de la interfaz de usuario y Accesibilidad.
    Se realizan pruebas de UX y de accesibilidad entre otras.
  • Eficiencia.- El sistema es rápido, escalable y estable incluso ante una gran concurrencia de usuarios, representa el desempeño relativo a la cantidad de recursos utilizados bajo determinadas condiciones. El sistema cumple Comportamiento temporal, Utilización de recursos y Capacidad.
    Se realizan pruebas de carga, resistencia, peak, etc.
  • Mantenibilidad.- El sistema debe ser capaz de ser modificado efectiva y eficientemente, debido a necesidades evolutivas, correctivas o perfectivas. Debido a aquello deber pensar en desarrollar de forma modular, en la reutilización, la capacidad de análisis, la modificabilidad y la capacidad de prueba.
    Las pruebas que se realizan son revisión de pares o el análisis estático.
  • Portabilidad.- El sistema debe moverse de un entorno a otro entorno hardware, software, operacional o de utilización con facilidad. El sistema debe cumplir con principios de Adaptabilidad, Capacidad para ser instalado y Capacidad para ser reemplazado.
    Se realizan pruebas para ver qué tan bien se puede instalar o desinstalar en un entorno específico.
  • Seguridad.- El sistema debe proteger la información y los datos de los usuarios para que personas o sistemas no autorizados no puedan leerlos o modificarlos, teniendo en cuenta los principios de confidencialidad, integridad, no repudio, responsabilidad y autenticidad.
    Se realizan pruebas de penetración, pruebas de vulnerabilidad, ethical hacking, y static analysis.
  • Compatibilidad.- El sistema debe ser compatible con su entorno (hardware y software), cumpliendo con la coexistencia,capacidad del producto para coexistir con otro software independiente; e interoperabilidad, capacidad de dos o más sistemas o componentes para intercambiar información y utilizar la información intercambiada.
    Se realizan pruebas de conformidad, compatibilidad, interoperabilidad y conversión.
